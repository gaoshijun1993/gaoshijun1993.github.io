hexo+GitHub Page 搭建个人博客踩坑指南

发表于 2017-04-10 | 分类于 前端 |

使用Github Page可以很方便快速的搭建属于自己的博客，关于这方面的介绍网上已经有相当多的教程可以参考。需要注意的一些地方是，Github Page的某些策略已经发生了改变，因此要参照网上最新的教程。推荐以下这两篇：

1.http://www.jianshu.com/p/834d7cc0668d
2.http://www.jianshu.com/p/169aa8765b3f

总的流程如下：

1.安装Node.js(如果已经安装，可略过）；
2.注册并安装Github客户端(如果已经安装，可略过）；
3.新建个人Repository，Clone到本地；
4.安装hexo；
5.使用hexo init命令初始化；
6.cd到项目文件夹，安装包依赖；
7.安装主题（可选）；
8.安装hexo-deployer-git；
9.部署项目到Github；

经过以上几个步骤，一个个人博客就算是搭建好了，虽然看起来略繁琐，如果熟悉的话操作起来相对还是很简单顺畅的。但是对于一个小白来说，还是很容易碰到一些坑的，我花了一天时间才算是搭建好，这里分享一下我踩的那些坑。

1.添加git命令到环境变量

如果在hexo init初始化的时候报错：

'git' 不是内部或外部命令，也不是可运行的程序或批处理文件。

则需要手动添加git到环境变量中，方法如下：

1.打开环境变量编辑面板，在“编辑用户环境变量”一栏中，选中变量为“PATH”的那一栏，点击编辑，点击新建，将Github安装路径中的以下连个路径添加进去。

usr\bin 和 libexec\git-core

以win10(64bit)为例，找到Github安装路径，路径分别如下

C:\Users\Administrator\AppData\Local\GitHub\PortableGit_f02737a78695063deace08e96d5042710d3e32db\usr\bin;
C:\Users\Administrator\AppData\Local\GitHub\PortableGit_f02737a78695063deace08e96d5042710d3e32db\mingw32\libexec\git-core;

2.代码部署到Github

如果在执行 hexo g -d后提示：

src refspec HEAD does not match any.

可能原因为当前还未创建分支，打开Github客户端，选中一个文件，尝试Commit操作，如果这个Commit成功，再进行hexo g -d部署操作。

注意：无须手动将本地代码Commit到Github中去，使用hexo g -d命令，hexo会自动将特定的文件Commit到Github中，手动Commit默认无法忽略node包依赖文件，导致Commit失败。

3.将Github Page绑定到你个人域名

虽然能很便宜的买到一些域名，但不花钱的域名还是更令人开心的，在http://www.dot.tk/zh/index.html可申请到一个免费的tk域名，并将此域名绑定到你的Github Page。

申请注册tk域名之后，按照教程ping你的Github Page得到你的Github Page ip
注意：得到的这个ip地址直接访问是指向一个404页面的，但是对绑定域名并不影响。
